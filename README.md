# Embedded_System_STM32
项目简介
电机控制是嵌入式系统的重要应用场景，本项目从基础驱动出发，逐步融合定时器PWM、串口通信、编码器测速、PID算法、状态机设计，打造了一款功能完备的智能直流电机控制系统。硬件采用STM32F103C8T6最小系统板、L9110S电机驱动模块、带霍尔编码器的直流减速电机，通过面包板搭建电路，验证各任务功能。
项目已完成所有STM32篇任务，代码结构清晰，支持双电机扩展（当前以电机1为主控对象），可通过串口指令灵活切换控制模式，并实时反馈转速信息。
硬件准备
• 主控：STM32F103C8T6开发板（带USB转串口）
• 电机驱动：L9110S模块（或类似H桥驱动）
• 电机：5V直流TT马达，带霍尔编码器（400线）
• 电源：5V电源模块（给电机供电），STM32由USB单独供电，共GND
• 其他：面包板、杜邦线、ST-Link/DAP烧录器

软件架构
项目采用模块化设计，各模块功能清晰：
• main.c：主循环，处理自动渐变模式、PID闭环控制、转速上报。
• encoder.c/h：编码器初始化、位置计数、转速计算（基于脉冲频率）。
• motor.c/h：电机PWM输出和方向控制。
• pid.c/h：PID算法实现，用于闭环速度调节。
• fsm.c/h：有限状态机，解析串口命令并管理电机状态。
• usart.c/h：串口接收中断处理、命令解析、字符串发送。
• tim.c/h：TIM1（PWM）、TIM3/TIM4（编码器）初始化。
• gpio.c/h：GPIO初始化（电机方向、其他外设）。
• stm32f1xx_hal_msp.c：HAL库硬件初始化回调。

使用说明
1. 编译下载：使用Keil MDK打开工程，选择芯片型号STM32F103C8，编译生成 .hex 文件，通过ST-Link下载至开发板。
2. 硬件连接：按照引脚定义连接电机驱动、编码器、串口。
3. 串口调试：打开串口助手（如PuTTY、SecureCRT），设置波特率115200、8N1，发送上述命令测试。
4. 观察现象：
◦ 发送 ON 后电机转动，发送 OFF 停止。
◦ 发送 SPEED 80 可改变转速。
◦ 发送 AUTO 后电机自动循环加减速。
◦ 发送 SET_RPM 300 进入PID闭环，串口每秒打印目标转速与实际转速，稳定后误差较小。
◦ 发送 RPM 查询当前转速。
◦ 发送 SELECT 2 可选择电机2（需硬件支持）。
参数配置
• PWM频率：1kHz（TIM1预分频71，周期999，72MHz系统时钟）
• 编码器每转脉冲数：ENCODER_PPR 宏定义在 encoder.h，默认400（可根据实际编码器修改）
• PID参数：在 main.c 中 PID_Init(&pid, 1.0f, 0.1f, 0.0f) 可调整
• PID更新周期：10ms（主循环通过定时器控制）
• 自动渐变步长：每30ms增加或减少5个PWM占空比单位
注意事项
• 电机驱动模块需单独供电，STM32与电机驱动模块必须共地（GND）。
• 编码器信号线建议使用上拉电阻或配置内部上拉，提高抗干扰能力。
• 若使用双电机，需确保TIM1的CH2也正确初始化，并在 motor.c 中配置对应的PWM通道。
• 默认代码仅启用电机1的PID控制，电机2的PID控制需在 main.c 中扩展。
